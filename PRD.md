# 产品需求文档 (PRD)：雅思单词智能听写与打字练习工具 (MVP版)

**文档版本:** V1.0 (MVP) **产品定位:** 一款将“听写诊断”、“错误分类复习”与“AI语境输出”相结合的雅思词汇闭环训练工具。 **目标用户:** 需要备考雅思，希望摆脱机械背词，提升拼写准确度、听音反应速度及英文打字速度的考生。

## 1. 产品背景与目标

传统背词软件多注重“认读”（看英文想中文），导致考生在听力考试中“听不懂、拼不对”，在写作中“想得出、打字慢”。 本项目旨在通过 MVP 版本验证以下闭环逻辑：

1.  **真实暴露弱点**：通过纯听写模式测试单词掌握度。

2.  **AI 智能诊断**：分析拼写错误特征，识别是“听音不准”、“拼写易混”还是“词汇变形”问题。

3.  **精准打击**：按错误类型分流到不同专项训练池。

4.  **语境与肌肉记忆**：利用大模型（LLM）将错词编织成雅思大作文，通过打字练习形成肌肉记忆和语境认知。

## 2. 核心业务流程 (User Flow)

`[选择词书/任务] -> [播放音频听写] -> [提交校验] -> [智能诊断分流] -> [错词进入专项池] -> [满足触发条件] -> [生成AI作文] -> [沉浸式打字练习] -> [数据报表]`

## 3. 功能需求详细说明

### 3.1 词库与任务模块

*   **内置词库：** MVP 阶段预置 300 个雅思高频听写词汇。每个词条包含：`word` (单词), `phonetic` (音标), `translation` (中文释义), `audio_url` (或使用浏览器 TTS)。

*   **每日任务：** 用户可设置每日听写数量（如 20 个/天）。

### 3.2 基础听写工作区

*   **音频播放：** 自动播放当前单词音频，支持快捷键（如 `Tab` 或 `Ctrl+Space`）重播。

*   **纯净输入：** 屏幕中央提供大字号输入框，支持回车键 (`Enter`) 快速提交。

*   **辅助提示（可选）：** 默认盲听。用户可点击“提示”按钮，依次解锁：词性 -> 首字母 -> 中文释义。使用提示即记为“未完全掌握”。

### 3.3 核心：智能错误诊断引擎 (后台逻辑)

当用户提交的拼写与目标单词不一致时，触发此引擎：

1.  **近音/词义混淆 (Sound-alike / Meaning Confusion):**

    *   **判定规则：** 用户输入的是合法英文单词（在词典库中），且发音编码（如 Metaphone 算法）与目标词高度相似。

    *   **处理：** 归入 **【近音辨析专项】**。

2.  **词缀变形错误 (Morphological Error):**

    *   **判定规则：** 用户输入的是合法英文单词，且提取词干（Stemming）后与目标单词词干一致（如 `analyze` vs `analysis`）。

    *   **处理：** 归入 **【语法变形专项】**。

3.  **视觉易错/拼写手误 (Typo / Visual Confusion):**

    *   **判定规则：** 用户输入的**不是**合法单词，且与目标词的编辑距离 (Levenshtein Distance) ≤ 2。

    *   **处理：** 归入 **【易混拼写专项】**。

4.  **完全生词 (Completely Unknown):**

    *   **判定规则：** 差距极大（编辑距离 > 2）、留空或直接查看答案。

    *   **处理：** 归入 **【生词重学专项】**。

### 3.4 专项复习模块

针对不同错误类型，提供不同的复习UI交互：

*   **近音辨析：** 展示目标词与错写词的对比卡片（包含两者释义和发音），要求用户进行二选一听音辨析。

*   **易混拼写：** 高亮显示拼错的字母，要求用户在输入框中连续正确“盲打” 3 次。

*   **生词重学：** 详细展示词根词缀、例句，进行常规复习。

### 3.5 终极巩固：AI 雅思作文打字模块

*   **触发条件：** 当错误池（重点是生词和拼写易错词）积累达到 10-15 个时解锁。

*   **AI 文本生成：**

    *   调用 LLM API（如 OpenAI / DeepSeek）。

    *   **Prompt：** `“请写一篇 250 词左右满足 7.5 分雅思 Task 2 大作文要求的文章。必须自然地包含以下词汇：[词汇列表]。请确保逻辑严密，并用 Markdown 加粗这些词汇。”“雅思写作达到7.5分需要满足：1.不仅回答问题，更要进行深入论述；2.分段合理，段落内及段落逻辑紧密，不仅仅是用firs,seconde,而是通过逻辑指代（This/These）和论证思路推动文章发展； 3. 展现出出色的同义转换能力，拼写和构词法错误极少；4. 能够熟练使用从句（状语、定语、名词性从句）、分词结构、虚拟语气、被动语态等，语法错误极少甚至没有。”`

*   **打字练习 (TypeRacer 模式)：**

    *   **展示区：** 显示生成的文章，生词高亮。

    *   **输入区：** 监听用户键盘输入，实时比对当前单词。正确显示绿色，错误显示红色并阻塞光标（需按退格键修改）。

    *   **统计：** 实时显示 WPM (词/分钟) 和 准确率 (Accuracy)。

## 4. 非功能需求

*   **技术栈建议：**前端 **Vue 3 (Vite 驱动) + TailwindCSS**。无需复杂后端，数据使用 `localStorage` 或 `IndexedDB` 存储。如果需要简单的 API 层来封装 LLM 调用，也可以考虑使用 Nuxt 3。

*   **API 依赖：** Web Speech API (免费发音)，LLM API (生成作文)。

*   **算法依赖：** npm 包 `natural` (用于 Metaphone 语音编码和词干提取) 和 `fast-levenshtein` (计算编辑距离)。
