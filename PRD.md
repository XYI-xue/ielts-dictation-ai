# 雅思单词智能听写与打字练习工具 (MVP版)

## 1. 产品目标与闭环

* **定位**

雅思词汇训练工具，把 **听写暴露弱点 → 错误类型诊断 → 分流复习 → 语境里打字巩固 → 看汇总** 连成闭环。

* **目标用户**

需要备考雅思，希望摆脱机械背词，提升拼写准确度、听音反应速度及英文打字速度的考生。

* **功能与目标**

传统背词软件多注重“认读”（看英文想中文），导致考生在听力考试中“听不懂、拼不对”，在写作中“想得出、打字慢”。 本项目旨在通过 MVP 版本验证以下闭环逻辑：

> * 真实暴露弱点：通过纯听写模式测试单词掌握度。
>
> * AI 智能诊断：分析拼写错误特征，识别是“听音不准”、“拼写易混”还是“词汇变形”问题。
>
> * 精准打击：按错误类型分流到不同专项训练池。
>
> * 语境与肌肉记忆：利用大模型（LLM）将错词编织成雅思大作文，通过打字练习形成肌肉记忆和语境认知。

* **核心价值**：

  > * 用 **听写 + 提交** 得到真实拼写错误，而不是只「认读」。
  >
  > * 用 **规则/算法** 把错误归到不同池子，复习交互不同。
  >
  > * 用 **LLM 生成范文** 把错词放进可打字的语境里，强化肌肉记忆与用法感知。

***

## 2. 核心业务流程

**[选择词书/任务] -> [播放音频听写] -> [提交校验] -> [智能诊断分流] -> [错词进入专项池] -> [满足触发条件] -> [生成AI作文] -> [沉浸式打字练习] -> [数据报表]**



可概括为：

1. **Dashboard** 看本轮/历史听写与错题概况；可看与「语境打字」同步的实时统计。

2. **听写**

   * 选本轮题量、排队练习。

   * **播放（TTS）→ 输入 → Enter 提交**。

   * 对：推进队列/轮次；错：**诊断分类 → 写入错题池（error log）→ 揭示/盲听等阶段**。

   * 轮次结束会归档会话，供首页与复习使用。

3. **专项复习**
   只消费 **听写来源** 的错题，按 **四类诊断** 映射到三个 Tab：

   * 近音辨析、拼写强化（含 typo + 词形）、生词重学。
     当前产品形态是 **单题卡片流 + 进度条 + 切 Tab**，每类有对应交互（二选一、盲打连对 3 次移出池、释义+默写等）。

4. **语境作文**

   * 选一轮听写（错词列表）→ 调 `deepseek大模型API`→ DeepSeek 返回 **英文范文 + 中文译文 + 好词好句**（JSON）。

   * 前端对范文做 **沉浸式逐字符打字**，并带 WPM/准确率/用时等统计。

## 3. 关键功能需求设计（按模块）

### 3.1 词库与任务模块

* **内置词库：** MVP 阶段预置 300 个雅思高频听写词汇。每个词条包含：`word` (单词), `phonetic` (音标), `translation` (中文释义), `audio_url` (或使用浏览器 TTS)。

* **每日任务：** 用户可设置每日听写数量（如 20 个/天）。

### 3.2 基础听写工作区

* **音频播放：** 自动播放当前单词音频，支持快捷键（如 `Tab` 或 `Ctrl+Space`）重播。

* **纯净输入：** 屏幕中央提供大字号输入框，支持回车键 (`Enter`) 快速提交。

* **辅助提示（可选）：** 默认盲听。用户可点击“提示”按钮，依次解锁：词性 -> 首字母 -> 中文释义。使用提示即记为“未完全掌握”。

* **听写轮次**：队列、题量、掌握判定、会话写入，并本地存储听写训练结果。



### 3.2 智能诊断与分流（核心后台逻辑）

| 设计意图      | 归类                          |
| :-------- | :-------------------------- |
| 近音/易混听辨   | `sound_alike` → 近音辨析        |
| 拼写手误/视觉易混 | `typo` → 拼写强化               |
| 词族/变形     | `typo` → 拼写强化               |
| 完全生词      | `completely_unknown` → 生词重学 |

* **听写提交错词时**：写入 `errorLogs`（含 `source: 'dictation'`, `recordedAt`），供复习与 Dashboard 统计。

* **规则设计：**

  * **近音/词义混淆 (Sound-alike / Meaning Confusion):**

    * **判定规则：** 用户输入的是合法英文单词（在词典库中），且发音编码（如 Metaphone 算法）与目标词高度相似。

    * **处理：** 归入 **【近音辨析专项】**。

  * **视觉易错/拼写手误 (Typo / Visual Confusion):**

    * **判定规则：**&#x20;

      * 用户输入的**不是**合法单词，且与目标词的编辑距离 (Levenshtein Distance) ≤ 2。

      * 用户输入的是合法英文单词，且提取词干（Stemming）后与目标单词词干一致（如 `analyze` vs `analysis`）。

    * **处理：** 归入 **【易混拼写专项】**。

  * **完全生词 (Completely Unknown):**

    * **判定规则：** 差距极大（编辑距离 > 2）、留空或直接查看答案。

    * **处理：** 归入 **【生词重学专项】**。

### 3.3 专项复习（差异化交互）

与 PRD 3.4 对齐的设计要点：

* **近音**：目标音 + 二选一（目标 vs 上次错写），可选默写巩固。

* **拼写强化**：对照正误 + **盲打连续对 3 次** → **移出错题池**。

* **生词**：词条+义+例句 + 默写自测（不必然移出池，偏验收）。

### 3.4 语境作文 + 打字（LLM 模型 API调用）

* 针对不同错误类型，提供不同的复习UI交互：

  * **近音辨析：** 展示目标词与错写词的对比卡片（包含两者释义和发音），要求用户进行二选一听音辨析。

  * **易混拼写：** 高亮显示拼错的字母，要求用户在输入框中连续正确“盲打” 3 次。

  * **生词重学：** 详细展示词根词缀、例句，进行常规复习。

* **触发**：**用户主动选轮次 + 生成**，更灵活。

* **打字练习作文生成**：服务端读一个写好的Prompts.md，规定好生成的要求、内容类型、返回格式等。

* **打字**：字符级 TypeRacer 式体验 + 统计；与 Dashboard 的同步属于展示层设计。

### 3.5 非功能需求

* **前端**：Vue 3 + Nuxt + Tailwind。

* **数据**：错题、会话等 **本地持久化**（localStorage 系工具），无账号则 **数据仅在浏览器**。

* **算法依赖**：`natural`、`fast-levenshtein`。

* **API**：LLM 走 **服务端路由**，密钥环境变量。

## 5. 原型设计

* Dashboard&#x20;

  * 布局：顶部导航 (Logo, Dashboard, Dictation, Review, Typing)

  * 内容：

    * 上方：三个大号 CTA 页面跳转按钮

    * 中间：四个数据统计卡片 (平均打字 WPM、今日需听写、练习轮数、待复习错词)

    * 下方：每轮练习错词统计详细信息

* Dictation

  * 布局： 极简深色/浅色背景，高度聚焦

  * 中心区域：一个巨大的播放图标 (可点击重播)。下方是一个 `text-4xl` (超大字体) 的输入框，只允许输入英文字母

  * 下方区域：

    * 进度条：显示本轮练习进度

    * 小卡片：显示目标单词信息以及出错提示

* Review

  * 布局：页面分为三个 Tab (近音辨析 / 拼写强化 / 生词重学)。

  * 近音辨析Tab的卡片：

    * 上方：音频播放按钮

    * 中间：左右两个单词选择卡片（单词 + 选择按钮）

    * 下方：输入框：拼写练习

  * 拼写强化Tab的 卡片：

    * 上方：单词信息卡片：错误拼写 VS 目标单词

    * 中间：输入框：拼写练习

  * 生词重学Tab的卡片：

    * 上方：错误拼写记录

    * 中间：目标单词信息

    * 下方：输入框：拼写练习

* Typing

  * 上方：展示生成的雅思大作文 (Task 2 Topic)。段落清晰。被选中的错词（如 **significant**, **analysis**）用强调色加粗显示。

  * 中间： 代码编辑器风格的打字区。

    * 已经打对的字：灰色/绿色

    * 当前正在打的字：带下划线，实时根据键盘输入变色

    * 打错的字：红色背景高亮

  * 下方：

    * 文本框：显示中文翻译

    * 卡片：展示好词好句积累建议。（英语句子+翻译+点评）

  * 浮窗：动态更新的统计数据面板 (WPM: 65, Accuracy: 97%, Time: 01:15)
